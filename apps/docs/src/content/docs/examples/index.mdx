## Running the Examples

### Prerequisites

Make sure you have the following installed:
- **Node.js** 18.0 or higher
- **pnpm** 8.0 or higher (recommended)
- **Git**

### Setup Instructions

You must run **2 projects**:
1. **Walrus Portal** - The blockchain portal service
2. **Site Builder SDK Playground** - The interactive playground

#### Setup Portal (https://docs.wal.app/walrus-sites/portal.html)

Portal configuration is managed through two key elements:
- **Environment variables**: Required for basic functionality
- **Constants file**: Optional for advanced customization

You can choose between **Server** or **Service Worker** deployment:

<Tabs syncKey="portal-type">
<TabItem label="Server Portal">

**1. Clone and set up repository**
```bash
git clone https://github.com/MystenLabs/walrus-sites.git
cd walrus-sites
git checkout mainnet
```

**2. Configure environment variables**
```bash
cp ./portal/server/.env.testnet.example ./portal/server/.env.local
```

**3. Install Bun (if not installed)**
```bash
# Check if bun is installed
bun --version

# If not installed, install it
curl -fsSL https://bun.sh/install | bash
# or
npm i -g bun
```

**4. Install dependencies**
```bash
cd portal
bun install
```

**5. Run server portal**
```bash
bun run server
```

The server portal will be available at **localhost:3001**

</TabItem>
<TabItem label="Service Worker Portal">

**1. Clone and set up repository**
```bash
git clone https://github.com/MystenLabs/walrus-sites.git
cd walrus-sites
git checkout mainnet
```

**2. Configure environment variables**
```bash
cp ./portal/worker/.env.testnet.example ./portal/worker/.env.local
```

**3. Install Bun (if not installed)**
```bash
# Check if bun is installed
bun --version

# If not installed, install it
curl -fsSL https://bun.sh/install | bash
# or
npm i -g bun
```

**4. Install dependencies**
```bash
cd portal
bun install
```

**5. Build and run service worker portal**
```bash
bun build:worker
bun run worker
```

The service worker portal will be available at **localhost:8080**

</TabItem>
</Tabs>

#### Setup Site Builder SDK Playground

This monorepo contains several applications and packages:

```
ts-sdks/
├── apps/
│   ├── docs/          # Documentation site (Port: 4321)
│   └── playground/    # Interactive playground (Port: 3001)
├── packages/
│   └── site-builder/  # Core SDK package
└── configs/           # Shared configurations
```

1. **Clone the repository**
   ```bash
   git clone https://github.com/CommandOSSLabs/ts-sdks
   cd ts-sdks
   ```

2. **Install dependencies**
   <Tabs syncKey="pkg">
   <TabItem label="pnpm">
   
   ```bash
   pnpm install
   ```
   
   </TabItem>
   <TabItem label="npm">
   
   ```bash
   npm install
   ```
   
   </TabItem>
   <TabItem label="yarn">
   
   ```bash
   yarn install
   ```
   
   </TabItem>
   </Tabs>

3. **Start development servers**
   <Tabs syncKey="pkg">
   <TabItem label="pnpm">
   
   ```bash
   # Start all services
   pnpm run dev
   
   # Or start individually
   pnpm --filter playground run dev    # Playground on port 3001 if portal is running first, otherwise port 3000
   pnpm --filter docs run dev          # Docs on port 4321
   ```
   
   </TabItem>
   <TabItem label="npm">
   
   ```bash
   # Start all services
   npm run dev
   
   # Or start individually
   npm --workspace=playground run dev  # Playground on port 3001 if portal is running first, otherwise port 3000
   npm --workspace=docs run dev        # Docs on port 4321
   ```
   
   </TabItem>
   <TabItem label="yarn">
   
   ```bash
   # Start all services
   yarn dev
   
   # Or start individually
   yarn workspace playground dev       # Playground on port 3001 if portal is running first, otherwise port 3000
   yarn workspace docs dev             # Docs on port 4321
   ```
   
   </TabItem>
   </Tabs>

:::note[Portal Configuration]
The `PORTAL_HOST` is set to `"localhost:3000"` by default in `apps/playground/src/app/page.tsx`. If you deploy using the **Service Worker Portal** (port 8080), you should update this configuration accordingly.
:::
